üìñ Welcome to QuestFoundry

QuestFoundry is a comprehensive specification for creating interactive branching narratives. Unlike traditional single-author tools, QuestFoundry defines a multi-agent studio where humans and AI can collaborate through structured workflows, validated data, and clear quality criteria.

Commands

üéØ Key Design Principles

1. Separation of Concerns

Each layer has a clear purpose:

  • What we do (Layers 0-1) ‚Äî Vision, roles, workflows
  • What we say (Layers 2-3) ‚Äî Data structures, schemas
  • How we communicate (Layer 4) ‚Äî Protocol, state machines
  • How we implement (Layers 5-7) ‚Äî Prompts, code, interfaces

2. Hot/Cold Architecture

Separates discovery from production:

  • Hot ‚Äî Discovery space, work-in-progress, experimentation
  • Cold ‚Äî Production canon, validated, immutable snapshots
  • Merge Gates ‚Äî Quality checks before Hot ‚Üí Cold promotion

3. Loop-Focused Workflows

13 canonical loops as single-source-of-truth procedures:

  • Discovery ‚Äî Story Spark, Hook Harvest, Lore Deepening
  • Refinement ‚Äî Codex Expansion, Style Tune-up
  • Assets ‚Äî Art Touch-up, Audio Pass, Translation Pass
  • Quality & Export ‚Äî Gatecheck, Binding Run, Narration Dry-Run, Post-Mortem, Archive

4. Quality Enforced

8 mandatory validation checks before production:

  • Integrity ‚Äî No dangling references, all sections reachable
  • Completeness ‚Äî Required fields present, metadata complete
  • Style ‚Äî Register consistency, voice alignment
  • Presentation ‚Äî No spoilers in player text, accessibility checks
  • Schema Conformance ‚Äî All artifacts validate against JSON schemas
  • Protocol Compliance ‚Äî Messages follow envelope v0.2.1 specification

5. Player Safety First

Player-Narrator (PN) sees only validated, spoiler-free content:

  • PN Boundary ‚Äî Only Cold content, never Hot
  • Spoiler Hygiene ‚Äî Enforced at protocol level via safe_for_pn field
  • Accessibility ‚Äî Built into quality bars and validation
